# Caffeine POS - การตั้งค่าฐานข้อมูล Supabase และการขึ้นระบบ (Deployment)

ยินดีต้อนรับสู่ระบบ POS ร้านกาแฟ Caffeine POS! เอกสารนี้แนะนำวิธีตั้งค่าตารางฐานข้อมูลใน Supabase, การเก็บ Code บน GitHub และการเอาแอปขึ้น Vercel

---

## 1. วิธีนำ Code ขึ้น GitHub
1. เปิด Git Bash หรือ Command Prompt ในโฟลเดอร์โปรเจกต์นี้
2. เริ่มใช้งาน Git และเชื่อมต่อกับ Repository ของคุณ:
   ```bash
   git init
   git add .
   git commit -m "Initial commit: Coffee Shop POS system"
   git branch -M main
   git remote add origin <ลิงก์-repository-ของคุณ-บน-github>
   git push -u origin main
   ```
> [!NOTE]
> ไฟล์ `.gitignore` ได้ถูกเพิ่มลงในโปรเจกต์นี้แล้ว เพื่อป้องกันไม่ให้ข้อมูลการตั้งค่าส่วนตัวหรือไฟล์ Log ต่างๆ ถูกอัปโหลดขึ้น GitHub

---

## 2. วิธีการเผยแพร่แอปบน Vercel (Deployment)
เนื่องจากแอปนี้เป็น **Static Web App (HTML, CSS, JS)** คุณสามารถเอาขึ้น Vercel ได้ฟรีและเสร็จสิ้นใน 1 นาที:

1. สมัครใช้งานและเข้าสู่ระบบ [Vercel](https://vercel.com)
2. กดปุ่ม **Add New...** -> เลือก **Project**
3. คัดเลือก Repository จาก GitHub ที่คุณเพิ่งอัปโหลดขึ้นไป
4. ในส่วน **Framework Preset** ให้ปล่อยเป็น **Other** หรือ **Vite** (เลือกเป็น Other สำหรับแอปปกติ)
5. ปล่อยการตั้งค่า Build Command และ Output Directory ไว้ตามปกติ (ไม่ต้องตั้งค่าอะไร)
6. กดปุ่ม **Deploy**
7. เมื่อสร้างเสร็จ Vercel จะมอบ URL สาธารณะ (เช่น `caffeine-pos.vercel.app`) ให้คุณใช้เข้าผ่านมือถือ แท็บเล็ต หรือเครื่อง POS ได้ทันที!

---

## 3. ขั้นตอนการตั้งค่าใน Supabase

1. สมัครใช้งานและสร้างโปรเจกต์ใหม่ใน [Supabase](https://supabase.com)
2. เข้าไปที่เมนู **SQL Editor** ใน Dashboard ของ Supabase
3. กดสร้าง Query ใหม่ (New Query) และคัดลอกโค้ด SQL ด้านล่างนี้ไปวาง
4. กดปุ่ม **Run** เพื่อสร้างตารางและใส่ข้อมูลเริ่มต้น

### คำสั่ง SQL สำหรับสร้างตารางและข้อมูลเริ่มต้น

```sql
-- 1. ตารางพนักงาน (employees)
create table public.employees (
  id uuid default gen_random_uuid() primary key,
  pin text not null unique,
  name text not null,
  role text not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 2. ตารางสินค้า (products)
create table public.products (
  id bigint generated by default as identity primary key,
  name text not null,
  category text not null, -- 'coffee' หรือ 'bakery'
  price numeric not null,
  icon text, -- อิโมจิสำหรับแสดงผล เช่น ☕, 🥐
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 3. ตารางลูกค้าสมาชิก (customers)
create table public.customers (
  id uuid default gen_random_uuid() primary key,
  name text not null,
  phone text not null unique,
  points integer default 0 not null,
  created_at timestamp with time zone default timezone('utc'::text, now()) not null
);

-- 4. ตารางประวัติธุรกรรมสั่งซื้อ (transactions)
create table public.transactions (
  id text primary key, -- รหัส TX-XXXXXX
  created_at timestamp with time zone default timezone('utc'::text, now()) not null,
  employee_name text not null,
  customer_name text,
  customer_phone text,
  subtotal numeric not null,
  discount numeric not null,
  total numeric not null,
  points_earned integer not null,
  points_redeemed integer not null,
  payment_method text not null,
  items jsonb not null -- รายการสินค้าทั้งหมดในตะกร้า
);

-- -----------------------------------------------------
-- ใส่ข้อมูลพนักงานเริ่มต้น (PIN สำหรับใช้ Login)
```

### การตั้งค่าความปลอดภัย (Row Level Security - RLS)

เพื่อความง่ายในการทดสอบและเชื่อมต่อแบบ Client-side แอปพลิเคชันนี้ทำงานผ่าน **Anon Key** เป็นหลัก คุณสามารถตั้งค่าปิด RLS ชั่วคราวบนตารางเหล่านี้เพื่อให้สามารถดึงข้อมูลได้ทันที:

```sql
alter table public.employees disable row level security;
alter table public.products disable row level security;
alter table public.customers disable row level security;
alter table public.transactions disable row level security;
```

## 4. การจัดการความปลอดภัยของข้อมูล
เนื่องจากแอปนี้เป็น Front-end แบบ 100% ตัวแอปพลิเคชันจะให้กรอก URL และ Key ของ Supabase ผ่านทางเมนู **"ตั้งค่าระบบ"** ในตัวโปรแกรมเอง โดยข้อมูลนี้จะถูกเก็บรักษาไว้ในบราวเซอร์ของแต่ละเครื่องเท่านั้น (ผ่าน LocalStorage) ทำให้ปลอดภัย ไม่เกิดการหลุดรั่วของคีย์บน GitHub แน่นอน
